# Welcome

Welcome to the DSA Roadmap — a structured, pattern-based approach to mastering Data Structures and Algorithms for coding interviews and competitive programming.

## What This Roadmap Is

This is not a random collection of LeetCode problems. It's a deliberate curriculum that:

- Introduces patterns in the right order (simpler → complex)
- Groups problems by the underlying technique, not just topic
- Provides hints that teach you *how to think*, not just what to code
- Covers every major pattern tested in FAANG and top-tier interviews

## What You'll Need

- Solid grasp of at least one programming language (Python, Java, C++, or JavaScript)
- Familiarity with basic syntax: loops, conditionals, functions, classes
- A LeetCode account (free tier is sufficient for most problems)
- Patience and consistency — this is a marathon, not a sprint

## Suggested Schedule

| Pace | Time per day | Estimated completion |
|------|-------------|----------------------|
| Intensive | 3–4 hours | 3–4 months |
| Moderate | 1–2 hours | 6–8 months |
| Casual | 30–60 min | 10–12 months |

## Mindset

- **Struggle is learning.** If a problem feels hard, that's the point.
- **Pattern recognition > memorization.** You're building intuition, not a lookup table.
- **Review your wrong answers.** Understanding why you were wrong is more valuable than getting it right.
- **Spaced repetition.** Revisit problems you found hard after 1 week, then 1 month.

## How Problems Are Structured

Each problem file contains:
1. **Problem Statement** — what the problem asks
2. **Examples** — concrete input/output pairs
3. **Constraints** — bounds that hint at expected complexity
4. **Hints** — three progressive nudges (try each before reading the next)
5. **Approach** — the optimal strategy in plain English (no code)
