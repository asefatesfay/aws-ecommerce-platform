# 31. Maths / Geometry

## Overview
This section covers number theory, integer manipulation, and computational geometry patterns that appear in interviews. These problems often require careful handling of edge cases, overflow, precision, and combinatorial counting.

## Key Concepts
- **Number basics**: digit operations, base conversion, divisibility
- **Prime and factor reasoning**: sieves, factor counting, trailing-zero math
- **Modular arithmetic**: large exponentiation and constraints with modulo
- **Geometry primitives**: distance, slope, area, collinearity, and triangle validity

## Common Patterns
| Pattern | Key Idea |
|---------|----------|
| Digit simulation | Process number/string one digit at a time |
| Math formula reduction | Replace brute force with derived formulas |
| Prime preprocessing | Use sieve for repeated primality/count queries |
| Geometry hashing | Canonical representations for lines/rectangles |
| Overflow-safe arithmetic | Use bounds checks and larger numeric types |

## When to Use
- Integer transform/reversal with strict bounds
- Counting outcomes under modulo constraints
- Coordinate sets requiring line/rectangle geometry checks
- Problems where direct simulation is too slow but formulas exist
