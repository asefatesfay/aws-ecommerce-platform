## How to Use This Repo

1. **Work in order** — sections build on each other. Arrays before Sliding Window, Trees before Graphs.
2. **Read the section README first** — understand the pattern before jumping into problems.
3. **Try each problem yourself** — spend at least 20–30 minutes before reading hints.
4. **Use hints progressively** — Hint 1 → attempt → Hint 2 → attempt → Hint 3.
5. **Check the Approach** — only after you've solved it or truly exhausted your attempts.
6. **Track your progress** — check off problems in the tracker above as you complete them.

## Pattern Recognition Tips

### Arrays & Strings
- Sorted array + searching → Binary Search or Two Pointers
- Subarray sum/count → Prefix Sum or Sliding Window
- In-place modification → think about index tricks and swapping

### Hash Tables
- "Find duplicate", "count frequency", "group by" → HashMap
- O(1) lookup requirement → always think HashMap first

### Two Pointers
- Sorted array + pair/triplet sum → Two Pointers from both ends
- Slow/fast pointer → cycle detection, middle of list
- Left/right shrinking window → partition problems

### Sliding Window
- Fixed-size subarray → fixed window
- "Longest/shortest subarray with condition" → variable window
- Shrink window when constraint violated, expand otherwise

### Binary Search
- Sorted array → obvious binary search
- "Minimum/maximum value that satisfies condition" → binary search on answer
- Monotonic function → binary search on the domain

### Trees
- Level-by-level → BFS with queue
- Path problems → DFS with recursion
- BST → leverage sorted property (left < root < right)

### Graphs
- Connected components → Union-Find or DFS/BFS
- Shortest path (unweighted) → BFS
- Shortest path (weighted) → Dijkstra
- Dependency ordering → Topological Sort

### Dynamic Programming
- "Count ways", "min/max cost", "is it possible" → likely DP
- Identify: state, transition, base case
- Top-down (memoization) vs bottom-up (tabulation) — both valid

### Greedy
- Local optimal = global optimal → Greedy
- Sorting + scanning → common greedy pattern
- Interval problems → sort by end time

- **Easy** — master the pattern, then these should take < 15 min
- **Medium** — the bulk of interviews; aim for 20–35 min
- **Hard** — combination of patterns; 45+ min is normal
